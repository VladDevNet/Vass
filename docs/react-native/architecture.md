# Архитектура

## Каркас

- **React Native + Expo (dev client / prebuild)** — Expo-инструментарий
  (сборка, OTA-обновления через EAS Update — аналог нынешнего «git pull и
  готово»), но с нативными модулями, которые понадобятся для аудио.
- Один кодовый репозиторий с бэкендом (монорепо: `mobile/` рядом с
  `VoiceAssistant.API/`).
- TypeScript.

## Правки бэкенда для мобильного клиента

Backend остаётся текущий .NET API на VPS (chat, память, grounding, интенты).
Мобильному клиенту нужны небольшие правки:

- Версионирование API (`/api/v1/...`) перед первым релизом приложения.
- Упрощённый вход для пожилых: device-token / магическая ссылка от
  родственника вместо email+пароль (настраивает член семьи).
- Напоминания — server-managed, locally scheduled: сервер разбирает голос и
  хранит каноническую запись, телефон планирует локальное уведомление через
  `expo-notifications`. Интернет для срабатывания не нужен; push остаётся
  возможным sync-hint, а не delivery-механизмом. См. [reminders.md](reminders.md).

Дальнейшие детали — в [BACKLOG.md](./BACKLOG.md), Фаза 0 и Фаза 4.

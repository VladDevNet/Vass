# Память: краткосрочная / среднесрочная / долгосрочная

Всё серверное (телефон — тонкий клиент), бэкенд уже наш.

## Краткосрочная (есть)

Скользящее окно последних сообщений: до 100 кандидатов, накопление с конца до
бюджета ~24000 символов (~6000 токенов). Реализовано в
`ChatController.Send()` (см. `MaxCandidateMessages` / `HistoryCharBudget`).

## Среднесрочная (новое)

Скользящее резюме сессии. Фоновая задача на Gemini: раз в N сообщений
сворачивает выпавшую из окна историю в компактное резюме, которое включается
в системный промпт. Хранение: `ChatSession.Summary`.

## Долгосрочная (новое): RAG на pgvector

Расширение к уже работающему Postgres:

- Извлечение фактов: параллельный side-call Gemini (паттерн уже отработан
  трижды: preamble, имена, инструкции) — вылавливает атомарные факты
  («лекарства лежат в верхнем ящике», «внук Миша приезжает в субботу») из
  каждой реплики.
- Таблица `MemoryFacts(id, userId, fact, embedding vector(768), createdAt,
  lastRecalledAt)`; эмбеддинги — Gemini `text-embedding-004` (дёшево, то же
  API).
- На каждый ход: эмбеддинг реплики → `ORDER BY embedding <=> query LIMIT 5` →
  top-K фактов в системный промпт.
- Голосовая команда «запомни ...» уже работает (`UserSettings.CustomSystemPrompt`)
  — остаётся для *поведенческих* предпочтений; *фактическая* память переезжает
  в RAG.

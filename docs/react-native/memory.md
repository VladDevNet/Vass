# Память: краткосрочная / среднесрочная / долгосрочная

Всё серверное (телефон — тонкий клиент), бэкенд уже наш.

## Краткосрочная (есть)

Скользящее окно последних сообщений: до 100 кандидатов, накопление с конца до
бюджета ~24000 символов (~6000 токенов). Реализовано в
`ChatController.Send()` (см. `MaxCandidateMessages` / `HistoryCharBudget`).

## Среднесрочная (есть)

Скользящее резюме сессии. Фоновая задача на Gemini: раз в N сообщений
сворачивает выпавшую из окна историю в компактное резюме, которое включается
в системный промпт. Хранение: `ChatSession.Summary`.

## Долгосрочная (есть): RAG на pgvector

Расширение к уже работающему Postgres:

- Извлечение фактов: параллельный side-call Gemini (паттерн уже отработан
  трижды: preamble, имена, инструкции) — вылавливает атомарные факты
  («лекарства лежат в верхнем ящике», «внук Миша приезжает в субботу») из
  каждой реплики.
- Таблица `MemoryFacts` хранит пользователя, атомарный факт, hash для точного
  дедупа, `embedding vector(768)`, embedding model, source message, даты и
  счётчик recall. Эмбеддинги — актуальный `gemini-embedding-2`; старый
  запланированный `text-embedding-004` больше не используется.
- На каждый ход: эмбеддинг реплики → `ORDER BY embedding <=> query LIMIT 5` →
  top-K фактов в системный промпт.
- Голосовая команда «запомни ...» уже работает (`UserSettings.CustomSystemPrompt`)
  — остаётся для *поведенческих* предпочтений; *фактическая* память переезжает
  в RAG.

Retrieval не вызывает embedding API, пока у пользователя нет ни одного факта.
Найденные top-5 ограничены cosine distance `0.45` и добавляются в системный
prompt как недоверенные данные, а не инструкции. Извлечение текущей реплики
идёт параллельно основному ответу на отдельном `DbContext`.

Пользовательский контроль:

- `GET /api/v1/memory/facts` — посмотреть сохранённые факты;
- `DELETE /api/v1/memory/facts/{id}` — удалить один;
- `DELETE /api/v1/memory/facts` — очистить всю долгосрочную память.

## Напоминания (следующий инкремент)

Хранилище и смысловой слой готовы, но delivery ещё нет: в мобильном клиенте
отсутствуют `expo-notifications`, push token registration и FCM/APNs. Поэтому
напоминания остаются незакрытым пунктом Фазы 4, а не имитацией через факт без
реального уведомления.

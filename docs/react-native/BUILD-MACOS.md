# Сборка и установка Vass на iPhone через Mac

Для того, у кого есть Mac, Claude Code и доступ к GitHub-репозиторию
проекта (добавлен как collaborator). Ставит приложение на **свой
собственный** iPhone (бесплатного Apple ID для этого достаточно — платный
Developer Account не нужен, он нужен только для раздачи чужим людям через
EAS).

Отдайте эту инструкцию своему Claude Code целиком — она написана так, чтобы
он мог провести через процесс сам, спрашивая вас только там, где нужны
физические действия с телефоном/Mac (подключить кабель, ввести Apple ID,
подтвердить диалог на экране).

## Что должно быть готово

- **Xcode** — из Mac App Store (бесплатно, но большой, ~10+ ГБ, если ещё не
  стоит — ставить заранее, до начала остального).
- **Node.js 20+** — если нет: `curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash`, затем `nvm install 22 && nvm use 22`.
- **Свой Apple ID** (обычный, бесплатный — тот же, что для App Store).
- **iPhone**, подключённый к Mac по USB (проще всего) или в той же Wi-Fi сети.
- **Доступ к репозиторию на GitHub** — должен быть выдан заранее (владелец
  добавляет вас как collaborator в настройках репозитория).

## 1. Склонировать репозиторий

```bash
git clone https://github.com/VladDevNet/Vass.git ~/vass
cd ~/vass/mobile
```

При первом обращении к приватному репозиторию git спросит авторизацию —
проще всего через `gh auth login` (GitHub CLI, если не стоит: `brew install gh`)
до клонирования, тогда HTTPS-клон пройдёт без лишних вопросов. Если git
просит логин/пароль напрямую — пароль GitHub давно не принимается, нужен
Personal Access Token (GitHub → Settings → Developer settings → Personal
access tokens) вместо пароля.

## 2. Установить зависимости

```bash
npm install
```

## 3. Собрать и установить на телефон

```bash
npx expo run:ios --device
```

Эта команда сама: сгенерирует нативный iOS-проект (папку `ios/`, если её ещё
нет), скомпилирует его, найдёт подключённый iPhone (если подключено
несколько устройств — спросит, какое), поставит приложение на него и
запустит локальный сервер разработки (Metro).

**Если на этом шаге попросит выбрать команду подписи (signing team) или
упадёт с ошибкой про code signing** — откройте сгенерированный проект в
Xcode и настройте вручную один раз:

```bash
open ios/*.xcworkspace
```

В Xcode: в списке слева выбрать проект → вкладка **Signing & Capabilities**
→ в поле **Team** выбрать свой Apple ID (если его нет в списке — меню
Xcode → Settings → Accounts → «+» → добавить Apple ID). После этого повторить
`npx expo run:ios --device`.

## 4. Разрешить запуск на самом iPhone

Первый запуск iOS заблокирует приложение как «непроверенное» — это
ожидаемо, разово на устройство:

1. **Настройки → Основные → VPN и управление устройством** → выбрать профиль
   с вашим Apple ID → **«Доверять»**.
2. **Настройки → Конфиденциальность и безопасность** → прокрутить до
   **«Режим разработчика»** → включить → телефон попросит перезагрузиться →
   после перезагрузки при разблокировке подтвердить диалог, ввести код-пароль.

После этого приложение должно открыться и работать — оно само обращается к
уже работающему серверу (`https://vass.it-consult.services`), никакой
дополнительной настройки бэкенда не нужно.

## Обновления

Когда код в репозитории изменится — просто `git pull` в `~/vass` и повторить
шаг 3 (`npx expo run:ios --device`). Пересобирать с нуля не нужно.

## Если что-то не совпадает с этой инструкцией

Expo часто меняет мелкие детали между версиями. Если команда/шаг ведёт себя
не так, как здесь описано — свежая документация под точную версию проекта:
https://docs.expo.dev/versions/v57.0.0/ (в `mobile/package.json` —
`"expo": "57.0.3"`).

## Ограничение

Приложение подписано вашим личным (бесплатным) Apple ID — такая подпись
перестаёт работать примерно через 7 дней, нужно будет переустановить
(повторить шаг 3) заново на свежую неделю.

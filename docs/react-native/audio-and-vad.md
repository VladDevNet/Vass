# Аудио-стек и распознавание (ядро продукта)

## Аудио-сессия

Нативный модуль поверх `AVAudioSession` (iOS, `playAndRecord` +
`voiceChat`-режим с эхоподавлением) и `AudioManager` (Android). Это снимает
всю категорию iOS-хаков веба: full-duplex запись+воспроизведение штатно
поддерживается ОС.

## VAD on-device: Silero VAD через onnxruntime

Модель ~2МБ, создана именно для этого. Замена самодельного RMS-порога:
различает речь и кухонный шум (требование презентации), устойчива к
расстоянию до микрофона (проблема, из-за которой пришлось приостановить
голосовую идентификацию — см. `SpeakerIdService` в бэкенде).

## Кнопки гарнитуры

`MPRemoteCommandCenter` (iOS) / MediaSession (Android) — play/pause как
push-to-talk / перебивание.

## Фоновый режим

iOS `audio` background mode (законный кейс — приложение непрерывно
пишет/воспроизводит), Android — foreground service с уведомлением.

## STT (распознавание речи)

Оставить **серверную Gemini-транскрипцию** (работает, качество подтверждено,
дёшево). Аудио шлём как сейчас (webm/m4a upload).

Опция на будущее (не в первых фазах): on-device Whisper (whisper.cpp small)
для офлайн-режима — отдельное исследование, не планируется раньше пилота.

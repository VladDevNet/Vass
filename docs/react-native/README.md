# Vass Mobile — React Native companion app

> Целевой продукт: голосовой ИИ-спутник для пожилых людей. Voice-first, без
> экранов и сложных интерфейсов, общение через беспроводные наушники. Тёплый
> собеседник + бытовой помощник + долговременная память. Это «Этап 2:
> Мобильный додаток» из презентации, расширенный до полноценного компаньона.

Этот каталог — рабочая документация мобильного приложения. Раньше всё лежало
в одном файле `PLAN-REACT-NATIVE.md`; теперь разбито по темам, чтобы читать
и обновлять можно было по частям.

## 0. Что уже доказано веб-прототипом (Этап 1)

Работает и переносится концептуально в RN:
- Полный голосовой цикл: VAD по RMS → запись → Gemini-транскрипция → LLM
  (Gemini 3.5 Flash + Google Search grounding) → стриминговый TTS (Piper, raw
  PCM) с проигрыванием по предложениям.
- Перебивание ассистента голосом, «дослушивание» продолжения фразы (shadow
  capture), фразы-заполнители при долгих операциях (preamble race).
- Реальный turn-taking: модель сама решает, закончил ли человек говорить или
  просто взял паузу подумать (потолок паузы 7с, backchannel-реплики).
- Голосовое управление поведением («запомни, говори медленнее») →
  постоянные инструкции в системном промпте.
- Ограничение краткосрочной памяти окном ~100 сообщений / ~6000 токенов.
- Экономика подтверждена: self-hosted TTS снял основную статью расходов,
  Gemini Flash даёт целевую себестоимость ~$1/мес на пользователя.

Больные места веба, которые обязано закрыть нативное приложение:
- iOS Safari аудио-хаки (разблокировка жестом, AudioContext interrupted,
  глушение микрофона при воспроизведении → не работает перебивание на iPad).
- Нет работы в фоне/при заблокированном экране.
- Нет интеграции с кнопками гарнитуры.
- Задержка сервера на TTS (сеть) вместо мгновенной локальной озвучки.

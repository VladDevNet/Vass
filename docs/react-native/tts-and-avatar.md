# Встроенный нейро-TTS и анимированный аватар

## TTS: требование — без сети, мгновенно

**Рекомендация: sherpa-onnx TTS** — рантайм с готовыми мобильными сборками
(iOS/Android), напрямую исполняет **те же Piper-модели**
(`ru_RU-irina-medium`, уже выбранный голос!) на устройстве через ONNX Runtime.
Модель ~60МБ в бандле, синтез быстрее реального времени на телефонных чипах.

- Плюс: голос идентичен текущему прод-голосу — переход незаметен.
- Запасной вариант A: системный `AVSpeechSynthesizer` / Google TTS (нулевая
  интеграция, но качество русских голосов хуже и различается между
  устройствами).
- Запасной вариант B: серверный Piper как fallback при проблемах на слабых
  устройствах (эндпоинт уже есть — `/api/chat/tts_stream`).

Стриминг по предложениям сохраняем: клиент режет ответ LLM по пунктуации
(логика из `frontend/js/yolo.js` — `extractCompleteSentences()` — переносится
1:1) и синтезирует локально каждое предложение — исчезает даже сетевой хоп
до VPS.

## Анимированный аватар

**Реализовано (2026-07-07, `mobile/src/components/AvatarFace.tsx`): обычное
2D-лицо на чистом `View` + `Animated`**, не Rive. Rive остаётся правильной
рекомендацией для качественного продакшн-варианта (векторные state machines,
60fps на слабых устройствах), но требует визуального редактора для авторства
`.riv`-файла (моргание, формы рта) — недоступен в среде, где пишется этот
код. Решение: пока обойтись полностью кодовым лицом (круги + прямоугольники,
анимация через `Animated.timing`/`interpolate`), вернуться к Rive когда
появится готовый `.riv`-асset.

Что есть сейчас:
- Голова-круг, глаза-круги, рот-прямоугольник — цвет головы меняется по
  состоянию (`idle`/`recording`/`thinking`/`speaking`).
- Моргание — независимый от голосового состояния таймер со случайным
  интервалом (2.2-4.8с), `scaleY` глаз 1 → 0 → 1. Работает всегда, не только
  во время разговора — иначе лицо выглядит «замороженным».
- Рот анимируется (`scaleY`-цикл) только в состоянии `speaking` — без
  привязки к реальной амплитуде TTS (буферизованный `/chat/tts` отдаёт файл
  целиком, а не поток, так что огибающей на лету пока нет — см. секцию TTS
  выше и `BACKLOG.md`).
- `thinking` — три пульсирующих точки под лицом (универсальный паттерн
  «печатает…»), а не взгляд вверх.
- Тап по лицу = старт/стоп записи (заменил отдельную emoji-кнопку).
- Проверено визуально через Expo web preview (скриншот) — единственная часть
  мобильного стека, которую реально можно увидеть без физического
  устройства, в отличие от аудио/VAD.

Что осталось от исходного плана Rive-варианта (не блокирует, доработка
позже): виземы/артикуляция по фонемам, привязка рта к реальной RMS-огибающей
(нужен потоковый, не буферизованный TTS), наклон/«внимание» на `listening`,
взгляд вверх на `thinking`.

Альтернатива (дороже, по-прежнему не рекомендуется): 3D-аватар
(react-three-fiber + Ready Player Me) — вес, батарея, сложность, см.
[risks.md](./risks.md).

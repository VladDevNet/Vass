# Встроенный нейро-TTS и анимированный аватар

## TTS: требование — без сети, мгновенно

**Реализовано: `expo-speech`** — системный TTS ОС
(`AVSpeechSynthesizer` на iOS, `TextToSpeech` на Android). Это единственный
runtime-путь голоса: в репозитории и на VPS нет серверного Piper- или
другого TTS-сервиса, а приложение не загружает модель синтеза в APK. Голос
на конкретном устройстве заранее неизвестен — он зависит от реально
установленных системных голосов (на Android обычно Google TTS).

Когда системный движок не инициализирован, не имеет подходящего голоса или
сам синтез завершается ошибкой, ответ остаётся в чате текстом, а голосовой
цикл возвращается к слушанию. Мы намеренно не подменяем это сетевой
генерацией: она делала сбой локального TTS причиной 10–80 секунд тишины и
держала пользователя в ложном состоянии «говорит».

Разобранные и отклонённые альтернативы (см. обсуждение с пользователем,
2026-07-07):
- **Silero** (пользователь считает более натуральным) — на устройстве
  недостижим: мейнтейнер отказался открывать ONNX-экспорт
  ([snakers4/silero-models#283](https://github.com/snakers4/silero-models/issues/283)),
  `react-native-pytorch-core`/PlayTorch (мост RN↔PyTorch) архивирован Meta.
  Реалистично только как отдельный будущий серверный продукт, не как
  fallback текущего мобильного приложения.
- **sherpa-onnx + Piper on-device** — работает технически, но увеличит APK
  и не гарантирует более натуральный русский голос; не входит в текущий
  релиз.
- **Kokoro** — официально не поддерживает русский.
- **MMS** (тот же sherpa-onnx-пайплайн) — около 102 МБ и без подтверждённого
  преимущества в натуральности для нашей аудитории.

Потоковый синтез по предложениям (как было в `frontend/js/yolo.js` —
`extractCompleteSentences()`, каталог удалён при
PROJECT-AUDIT-2026-07-10 ARCH-01, восстановим из истории git) для
`expo-speech` не нужен — синтез и так мгновенный (системный движок),
сетевого хопа нет независимо от стриминга.

## Анимированный аватар

**Реализовано (2026-07-07, `mobile/src/components/AvatarFace.tsx`): обычное
2D-лицо на чистом `View` + `Animated`**, не Rive. Rive остаётся правильной
рекомендацией для качественного продакшн-варианта (векторные state machines,
60fps на слабых устройствах), но требует визуального редактора для авторства
`.riv`-файла (моргание, формы рта) — недоступен в среде, где пишется этот
код. Решение: пока обойтись полностью кодовым лицом (круги + прямоугольники,
анимация через `Animated.timing`/`interpolate`), вернуться к Rive когда
появится готовый `.riv`-асset.

Что есть сейчас:
- Голова-круг, глаза-круги, рот-прямоугольник — цвет головы меняется по
  состоянию (`idle`/`recording`/`thinking`/`speaking`).
- Моргание — независимый от голосового состояния таймер со случайным
  интервалом (2.2-4.8с), `scaleY` глаз 1 → 0 → 1. Работает всегда, не только
  во время разговора — иначе лицо выглядит «замороженным».
- Рот анимируется (`scaleY`-цикл) только в состоянии `speaking` — без
  привязки к реальной амплитуде системного TTS. Для настоящей огибающей
  потребовалась бы отдельная native-интеграция.
- `thinking` — три пульсирующих точки под лицом (универсальный паттерн
  «печатает…»), а не взгляд вверх.
- Тап по лицу = старт/стоп записи (заменил отдельную emoji-кнопку).
- Проверено визуально через Expo web preview (скриншот) — единственная часть
  мобильного стека, которую реально можно увидеть без физического
  устройства, в отличие от аудио/VAD.

Что осталось от исходного плана Rive-варианта (не блокирует, доработка
позже): виземы/артикуляция по фонемам, привязка рта к реальной RMS-огибающей
(нужен потоковый, не буферизованный TTS), наклон/«внимание» на `listening`,
взгляд вверх на `thinking`.

Альтернатива (дороже, по-прежнему не рекомендуется): 3D-аватар
(react-three-fiber + Ready Player Me) — вес, батарея, сложность, см.
[risks.md](./risks.md).

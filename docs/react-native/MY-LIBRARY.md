# Моя библиотека

## Назначение

«Моя библиотека» хранит на телефоне созданные ассистентом книги и отчеты:
рецепты, подборки ресторанов, развлечений и произвольные гиды. Это не
серверное хранилище: Vass не сохраняет разметку книги, ее текст или версии
в базе API.

## Пользовательский сценарий

- Пользователь естественной фразой просит собрать или обновить книгу,
  например: «Сделай красивую книгу с ужинами на неделю».
- Центральная модель выбирает `library_write`, формирует статический HTML/CSS
  и коротко подтверждает действие.
- После окончания озвучивания клиент сохраняет artifact, выводит Vass из
  overlay на полный экран и открывает созданную книгу.
- Через «Настройки -> Моя библиотека» доступно ручное оглавление, удаление
  книги и выбор любой сохраненной версии.
- По голосовой просьбе «Открой мою книгу рецептов» модель получает только
  компактный каталог и вызывает `library_open` для нужного artifact.

## Локальный контракт

Файлы находятся в app-private Documents-каталоге `vass-library`:

- `index.v1.json` содержит только служебные metadata: UUID, название, тип,
  краткое описание, sources, дату и список revisions;
- каждая revision хранится отдельным HTML-файлом;
- одна книга хранит до 50 последних версий;
- в запрос ассистента попадает не более 20 элементов каталога, без HTML.

Клиент отправляет `supportsLibrary` только в версиях, которые умеют принять
инструмент. Старый клиент продолжает работать: сервер не предлагает ему
`library_write` и `library_open`.

## Модельные инструменты

`library_list` возвращает каталог, когда выбор книги неоднозначен.
`library_write` создает новую книгу или следующую версию существующей по
известному UUID. `library_open` открывает каталог либо конкретную книгу.

Сервер валидирует идентификаторы, тип, размер и статичность разметки, создает
только content-free `ActionReceipt`, а HTML передает клиенту единым SSE action
envelope. Receipt не содержит HTML и не является копией документа.

## Граница безопасности

Клиент повторно санитизирует HTML перед записью. Удаляются скрипты, формы,
iframe, медиа, SVG, event handlers, URL-атрибуты, inline styles, CSS `url()`
и `@import`. Reader использует `react-native-webview` без JavaScript, DOM
storage, native bridge, file access, cache и разрешенной навигации. Wrapper
добавляет CSP с `default-src 'none'`. Внешние ссылки и произвольный network
access отсутствуют.

Это defense in depth: модель не должна генерировать активный код, сервер
отвергает его до SSE, а reader не дает ему выполниться, даже если artifact
поврежден локально.

## Проверка релиза

1. Попросить создать книгу с рецептами и убедиться, что Vass сам открывает ее.
2. В настройках открыть оглавление, затем первую и последнюю revision.
3. Попросить обновить найденную книгу и проверить рост числа версий.
4. В overlay попросить открыть библиотеку и проверить возврат в полноэкранный
   Vass.
5. Удалить книгу вручную; повторная голосовая команда не должна открывать
   удаленный UUID.

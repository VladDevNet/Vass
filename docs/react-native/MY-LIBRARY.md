# Моя библиотека

## Назначение

«Моя библиотека» хранит на телефоне созданные ассистентом книги и отчеты:
рецепты, подборки ресторанов, развлечений и произвольные гиды. Книги лежат
в одном пользовательском разделе, а не в наборе жестких тегов. Это не
серверное хранилище: Vass не сохраняет разметку книги, ее текст или версии
в базе API.

## Пользовательский сценарий

- Пользователь естественной фразой просит собрать или обновить книгу,
  например: «Сделай красивую книгу с ужинами на неделю».
- Центральная модель выбирает `library_write`, формирует статический HTML/CSS
  и коротко подтверждает действие.
- После окончания озвучивания клиент сохраняет artifact, выводит Vass из
  overlay на полный экран и открывает созданную книгу.
- В «Настройки -> Моя библиотека» сначала открывается оглавление разделов,
  затем список книг выбранного раздела. После просмотра книги кнопка «назад»
  возвращает в её раздел.
- Пользователь может создать, переименовать или удалить раздел, а также
  вручную переместить книгу. При удалении непустого раздела книги остаются на
  устройстве и переходят в «Без раздела».
- По голосовой просьбе «Открой мою книгу рецептов» модель получает только
  компактный каталог разделов и книг и вызывает `library_open` для нужного
  artifact.

## Локальный контракт

Файлы находятся в app-private Documents-каталоге `vass-library`:

- `index.v2.json` содержит только служебные metadata разделов и книг: UUID,
  название, тип, раздел, краткое описание, sources, дату и список revisions;
- при первом запуске v2 старый `index.v1.json` читается без потери данных и
  переносится в v2. Старые книги автоматически получают разумный раздел по
  своему типу: рецепты переходят в «Кулинарию», гиды в «Путешествия и гиды»;
- каждая revision хранится отдельным HTML-файлом;
- одна книга хранит до 50 последних версий;
- в запрос ассистента попадает не более 30 названий разделов и 20 элементов
  каталога, без HTML.

Клиент отправляет `supportsLibrary` только в версиях, которые умеют принять
инструмент. Старый клиент продолжает работать: сервер не предлагает ему
`library_write` и `library_open`.

## Модельные инструменты

`library_list` возвращает разделы и каталог, когда выбор книги неоднозначен.
`library_write` создает новую книгу или следующую версию существующей по
известному UUID. Для новой книги модель передает `sectionTitle`: повторяет
подходящее пользовательское название раздела или формирует новое короткое
понятное имя. `library_open` открывает оглавление либо конкретную книгу.

Разделы не являются taxonomy или набором заранее заданных меток. Модель
использует их как обычные папки, а пользователь всегда может изменить
структуру вручную.

Сервер валидирует идентификаторы, тип, размер и статичность разметки, создает
только content-free `ActionReceipt`, а HTML передает клиенту единым SSE action
envelope. Receipt не содержит HTML и не является копией документа.

## Граница безопасности

Клиент повторно санитизирует HTML перед записью. Удаляются скрипты, формы,
iframe, медиа, SVG, event handlers, URL-атрибуты, inline styles, CSS `url()`
и `@import`. Reader использует `react-native-webview` без JavaScript, DOM
storage, native bridge, file access, cache и разрешенной навигации. Wrapper
добавляет CSP с `default-src 'none'`. Внешние ссылки и произвольный network
access отсутствуют.

Это defense in depth: модель не должна генерировать активный код, сервер
отвергает его до SSE, а reader не дает ему выполниться, даже если artifact
поврежден локально.

## Проверка релиза

1. Открыть существующую библиотеку после обновления и убедиться, что рецепты
   находятся в «Кулинарии».
2. Создать и переименовать произвольный раздел, затем вручную переместить в
   него книгу.
3. Попросить создать книгу с рецептами и убедиться, что Vass сам открывает
   ее, а кнопка «назад» возвращает в выбранный раздел.
4. Попросить обновить найденную книгу и проверить рост числа версий без
   самовольного изменения её раздела.
5. В overlay попросить открыть библиотеку и проверить возврат в полноэкранный
   Vass.
6. Удалить непустой раздел и убедиться, что книги переходят в «Без раздела».

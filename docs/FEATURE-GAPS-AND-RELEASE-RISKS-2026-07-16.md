# Найденные пробелы и release-риски

Статус: release candidate от 2026-07-16. Это рабочий чек-лист для
физической Android-проверки, а не список обещаний. Полный каталог
реализованных возможностей находится в
[`FEATURE-CAPABILITIES-AUDIT-2026-07-16.md`](./FEATURE-CAPABILITIES-AUDIT-2026-07-16.md).

## Что закрыто в этом инкременте

- При `REGISTRATION_AUTO_APPROVE=false` мобильный клиент показывает
  отдельный экран ожидания, а API возвращает понятный `approval_required`.
- Модель получила native function tools `reminder_list`, `reminder_cancel`
  и `capability_help`; в настройках есть экраны списка напоминаний и
  возможностей Vass.
- Долгосрочная память теперь имеет стабильные категории и может хранить
  ссылку на явно сохранённое изображение или документ. Вложения доступны из
  экрана памяти: изображение открывается внутри Vass, документ передаётся
  системному просмотрщику.
- После share и успешного screen capture Vass возвращается на передний план
  и произносит локальное подтверждение. Само подтверждение не уходит в
  модель как голосовой ввод.
- Legacy `PATCH`/`DELETE /chat/sessions/{id}` удалены: продукт сознательно
  остаётся одним непрерывным диалогом, а не многодиалоговым мессенджером.
- Server-side Piper TTS и его compose-сервис удалены. При сбое системного
  TTS ответ остаётся текстом, а разговор не ждёт сетевой fallback.

## Release-блокеры

Перед распространением APK должны быть пройдены все пункты ниже на реальном
Android ARM64-устройстве. Один успешный unit/integration test этого не
заменяет.

1. **Закрытая регистрация.** На стенде с `REGISTRATION_AUTO_APPROVE=false`
   зарегистрировать новый email, увидеть экран ожидания, затем подтвердить
   учётную запись из `/admin/` и войти тем же паролем.
2. **Share cold/warm start.** Отправить в Vass отдельно ссылку, обычный
   текст, изображение и PDF из другого приложения. Каждый раз Vass должен
   выйти на передний план, назвать правильный тип материала и показать
   ожидающее вложение до голосовой задачи.
3. **Screen capture.** Сказать «сделай снимок экрана», подтвердить Android
   consent, проверить возврат в полноэкранный Vass, фразу «Снимок экрана
   получен» и последующий ответ модели именно по прикреплённому кадру.
   Отдельно проверить отказ/отмену consent: разговор должен вернуться в
   нормальный listening без зависания и пустого вложения.
4. **Память с документом.** Расшарить ссылку/изображение/PDF, явно сказать
   «запомни», затем открыть «Память». Запись должна иметь осмысленный текст,
   корректную категорию, доступное вложение и после удаления записи не
   блокировать удаление исходного attachment.
5. **Напоминания.** Создать одноразовое и периодическое напоминание,
   убедиться в локальном срабатывании без сети, отменить одно голосом и одно
   в UI. Ни одно отменённое уведомление не должно сработать после
   перезапуска приложения.
6. **Voice/overlay/media.** Провести не менее 30 минут: полный экран,
   overlay, pause/resume, YouTube hand-off, возврат в Vass, перебивание и
   несколько TTS-реплик после внешнего аудио. Не должно быть «немого
   speaking», пропавшего микрофона, погасшего экрана во время активного
   диалога или застрявшего overlay.

## Существенные, но не блокирующие риски

| Риск | Текущее поведение | Решение до масштабирования beta |
| --- | --- | --- |
| Реальные форматы документов | Share/picker принимают один файл до 50 MiB; модель честно разбирает его best effort. Не каждый MIME можно прочитать или объяснить. | Сформировать compatibility matrix по результатам Android-сессий и добавить byte-level validation для поддержанных форматов. |
| Несколько файлов | `ACTION_SEND_MULTIPLE` и picker используют только первый URI. | Оставить честное ограничение в help, затем спроектировать очередь вложений. |
| Просмотр документов | Нужен внешний Android viewer для конкретного MIME/PDF. | Показать понятное сообщение, если handler отсутствует; встроенный PDF viewer рассматривать отдельно. |
| Категории памяти | 29 стабильных категорий плюс `other`; модель выбирает их семантически. | После beta-данных проверить фактическое распределение и объединить редкие/непонятные категории, не меняя стабильные IDs без миграции. |
| Подсказки возможностей | Есть каталог и экран справки, но ещё нет ненавязчивого just-in-time onboarding/progress telemetry. | Ввести локальный `CapabilityGuideProgress`, лимит инициативных подсказок и явный переключатель подсказок. |
| Скрытые постоянные инструкции | Модель может обновлять `CustomSystemPrompt`, но в mobile UI нет просмотра и отмены. | Не расширять это поведение; добавить прозрачный экран управления до публичного продвижения функции. |
| Использование моделей и токенов | Admin уже показывает пользователей/активность, но не metering токенов и стоимости. | Добавить content-free usage records до расширения числа beta-пользователей. |

## Границы, которые нельзя обещать пользователю

- Vass не читает экран постоянно и не получает кадр без системного consent.
- Модель не открывает камеру, галерею, chooser или Android permissions сама:
  она только объясняет путь, пользователь нажимает системный UI.
- YouTube tool подтверждает передачу запроса в приложение/браузер, а не
  фактическое воспроизведение ролика.
- Локальные напоминания планирует ОС телефона; сервер хранит intent и
  delivery state, но не способен разбудить выключенный/ограниченный Android.
- Vass не поддерживает несколько независимых историй чата в текущем
  companion-релизе.
